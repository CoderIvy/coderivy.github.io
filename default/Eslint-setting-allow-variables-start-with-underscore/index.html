Eslint setting allow variables start with underscore

By Lan Chen / 2019-07-23

I’ve got an Unexpected dangling '_' in '_origin' no-underscore-dangle error when I used _ as the beginning of a variable because of naming conventions for identifiers in JavaScript.

This error comes from rules 'no-underscore-dangle': ['error', { allow: ['_store'] }] which setuped in a setting file .eslintrc.js. As this variable is a “private” member of an object so I want to continue using “_” as the beginning of that variable. Of course, the easiest way to solve this problem is to cancel that rules in .eslintrc.js. But I don’t want to allow all variables can start with _.

There is an easy way to do this, adding this ignore comment /* eslint no-underscore-dangle:0 */ at the top of that variable. For example

 /* eslint no-underscore-dangle:0 */
 const origin = item[0]._origin;

In this way it will just ignore the error shows on that line.



Any feedback is welcome! :-)
