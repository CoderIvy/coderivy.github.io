Git clone authentication failed


Description:
Use “git clone my_repository_url” to clone a project form remote to local in Windows 10 Pro’s Terminal.

Result:
I’ve got an “Authentication failed” error.

Reason:
Git didn’t ask me to input my username and password for this repository, just simply show a failed error. And there are no credential details about this repository in windows credential manager.

There are many ways to solve this problem, I’ve tried the following 2 methods:

  1. use GitHub desktop to clone this project, it will auto pop-up a window remind you to input username and password. This method works for this problem.

  2. manually add git authentication to windows credential manager:
    open Credential Manager


click Windows Credentials



click ‘Add a Generic Credential’ in Generic Credentials input your repository server URL, username, and password (Note: it is server URL, not full project URL)



Important: add ‘git:’ before your server URL, the final result should be like: git:http://your_server_url




After adding this generic credential, you don’t have to input the username & password for all repositories form this server.
